01 · Teoria da Computação

Máquina de Turing

Modelo matemático proposto por Alan Turing em 1936 que define o limite do que é computável.

Definição Formal — 7-tupla
M = (Q, Σ, Γ, δ, q₀, q_aceita, q_rejeita)
Q = estados | Σ = alfabeto de entrada | Γ = alfabeto da fita (inclui ⊔) | δ: Q×Γ → Q×Γ×{E,D}
SímboloSignificadoExemplo
QConjunto finito de estados{q0,q1,q_ac,q_rej}
ΣAlfabeto de entrada{a,b}
ΓAlfabeto da fita{a,b,X,⊔}
δFunção de transiçãoQ×Γ→Q×Γ×{E,D}

Exemplo — Reconhecer aⁿbⁿ

Fita: [a][a][b][b][⊔] Estado Lê Escreve Mov Vai para q0 a X D q1 ← risca 'a' q0 X X D q0 ← pula X q0 ⊔ ⊔ — q_aceita ← só X restou! q1 a a D q1 ← anda direita q1 b X E q2 ← risca 'b' q2 a a E q2 ← volta q2 X X D q0 ← reinicia
💡
Tese de Church-Turing

Tudo que é efetivamente computável por um algoritmo pode ser computado por uma MT. O Problema da Parada é indecidível — não existe MT que determine se outra MT para.

03 · Complexidade Computacional

Classes P, NP e NP-Completo

Classe P
Problemas resolvíveis em tempo polinomial O(nᵏ)
Exemplos: ordenação, busca, caminho mínimo, teste de primalidade (AKS).
Classe NP
Problemas cuja solução pode ser verificada em tempo polinomial
P ⊆ NP. NP não significa "difícil" — significa que verificar é fácil, resolver pode não ser.
P vs NP — Problema do Milênio (US$1.000.000)

Ainda em aberto. A maioria acredita que P ≠ NP. NP-Completo = NP-Difícil ∩ NP. Exemplos NP-Completo: SAT, 3-SAT, Clique, TSP, Mochila 0/1.

04 · Linguagens Formais

Gramática Formal

Definição — 4-tupla
G = (V, T, P, S)
V = variáveis (não-terminais) | T = terminais | P = regras de produção α→β | S = símbolo inicial

Não-terminais (V)

Letras maiúsculas (S, A, B). Podem ser expandidos por regras.

Terminais (T)

Letras minúsculas (a, b, 0, 1). Não expansíveis — aparecem na cadeia final.

G = ({S}, {a,b}, P, S)  — Gramática para aⁿbⁿ
P = { S → aSb,  S → ab }

Derivação de "aaabbb":
S ⇒ aSb ⇒ aaSbb ⇒ aaabbb ✓

05 · Estruturas de Dados

Pilha (Stack) & Alocação Dinâmica

Estrutura LIFO — último a entrar, primeiro a sair. Todas as operações são O(1).

PUSH(A) → PUSH(B) → PUSH(C) → POP(): ┌───┐ ┌───┐ ← topo ┌───┐ ← topo │ B │ ← topo │ C │ │ B │ ├───┤ ├───┤ ├───┤ │ A │ │ B │ │ A │ └───┘ ├───┤ └───┘ │ A │ Retornou: C └───┘
class Pilha:
    def __init__(self): self._d = []
    def push(self, v): self._d.append(v)
    def pop(self):  return self._d.pop()
    def peek(self): return self._d[-1]

# Verificar parênteses balanceados:
def balanceado(s):
    p = Pilha(); pares = {')':'(',']':'[','}':'{'}
    for c in s:
        if c in '([{': p.push(c)
        elif c in ')]}':
            if p.empty() or p.pop() != pares[c]: return False
    return p.empty()

06 · Estruturas de Dados

Tabela Hash

Função hash básica
h(k) = k mod m
m = tamanho da tabela. Ex: m=7 → h(22)=1, h(31)=3. Colisão quando h(k₁)=h(k₂).
Fator de carga
α = n/m → manter α < 0.75, senão rehashing

Chaining

Lista ligada em cada célula. O(1) médio, O(n) pior.

Open Addressing

h(k,i)=(h(k)+i) mod m. Sem listas extras.

Double Hashing

h(k,i)=(h₁(k)+i·h₂(k)) mod m. Reduz clustering.

10 · Sistemas Operacionais

Memória Virtual & Paginação

Tradução de Endereço
Físico = frame_base + (virtual mod tamanho_página)
Ex: página=1KB, endereço=2500 → página=2, offset=452. Frame 1 → físico=1024+452=1476
Tempo Efetivo de Acesso
T = (1−p)·T_mem + p·T_page_fault
p=taxa de page faults. T_pf≈8ms, T_mem≈200ns. p=0.1% degrada 40× o desempenho!
AlgoritmoPolítica

20 · Matemática

Probabilidade & Teorema de Bayes

Probabilidade Condicional
P(A|B) = P(A∩B) / P(B)
Teorema de Bayes
P(A|B) = P(B|A)·P(A) / P(B)
Adição
P(A∪B) = P(A) + P(B) − P(A∩B)
Bayes — Teste médico:
P(doença)=0.001, P(+|doença)=0.99, P(+|saudável)=0.05

P(+) = 0.99×0.001 + 0.05×0.999 = 0.05094

P(doença|+) = 0.99×0.001 / 0.05094 ≈ 1.94%
Mesmo positivo, só ~2% de chance! Doença é rara.

from collections import deque
def bfs(g, s):
    vis={s}; fila=deque([s])
    while fila:
        v=fila.popleft(); print(v,end=' ')
        for u in g[v]:
            if u not in vis: vis.add(u); fila.append(u)

def dfs(g, v, vis=None):
    if vis is None: vis=set()
    vis.add(v); print(v,end=' ')
    for u in g[v]:
        if u not in vis: dfs(g,u,vis)

Logaritmos

Logaritmo responde: "que potência preciso para chegar nesse número?"

Definição
logₐ b = x ↔ aˣ = b (a≠1, a>0, b>0)
log₂ 8 = 3 porque 2³=8 | log₁₀ 1000=3 porque 10³=1000 | logₐ 1=0 porque a⁰=1
PropriedadeFórmulaExemplo numérico

🔗
Conexão com O(log n)

Na busca binária, cada passo divide o problema pela metade. 1.024 itens → log₂ 1024 = 10 passos. 1 bilhão de itens → log₂ 10⁹ ≈ 30 passos. Isso é o poder do logaritmo!

02 · Fundamentos

Iteração vs Recursão

🔁

Iterativo

Usa for/while. Memória O(1) — uma variável acumuladora. Mais rápido, sem risco de Stack Overflow.

🔄

Recursivo

Função chama a si mesma. Memória O(n) — cria pilha de chamadas. Mais elegante, risco de Stack Overflow.

Pilha de chamadas de fatorial(4): fatorial(4) chama fatorial(3) fatorial(3) chama fatorial(2) fatorial(2) chama fatorial(1) fatorial(1) → retorna 1 ← CASO BASE fatorial(2) → retorna 2×1 = 2 fatorial(3) → retorna 3×2 = 6 fatorial(4) → retorna 4×6 = 24 4 frames na pilha = O(n) memória
Regra de Ouro

"Todo algoritmo recursivo tem um equivalente iterativo." A recursão usa mais memória (pilha), mas o código tende a ser mais legível para problemas naturalmente recursivos.

03 · Algoritmos Clássicos

Fibonacci

Sequência: 0, 1, 1, 2, 3, 5, 8, 13, 21… cada número = soma dos dois anteriores.

Definição
f(0)=0, f(1)=1, f(n) = f(n-1) + f(n-2) para n≥2

Versão Iterativa — O(n) tempo, O(1) espaço

int a = 0, b = 1;
for (int i = 0; i < n; i++) { // O(n)
    System.out.print(a + " "); // O(1)
    int temp = a;
    a = b;
    b = temp + b;              // O(1)
}
// Total: O(n) | Espaço: O(1) — só a, b, temp
def fib_iter(n):
    a, b = 0, 1
    for _ in range(n):    # O(n)
        print(a, end=" ")
        a, b = b, a + b  # O(1) — troca simultânea!
# Total: O(n) | Espaço: O(1)
a ← 0; b ← 1
para i de 0 até n-1 faça   // O(n)
    imprimir(a)
    temp ← a; a ← b; b ← temp + b
fim-para
// O(4n + constantes) = O(n)

Versão Recursiva — O(2ⁿ) tempo, O(n) espaço

⚠️
Por que O(2ⁿ)?

Cada chamada gera 2 chamadas menores. fib(5)=15 chamadas. fib(30)>1 bilhão! Subproblemas recalculados repetidamente.

Árvore de chamadas de fib(4): fib(4) ├── fib(3) │ ├── fib(2) │ │ ├── fib(1) → 1 │ │ └── fib(0) → 0 │ └── fib(1) → 1 └── fib(2) ← calculado de NOVO! desperdício ├── fib(1) → 1 └── fib(0) → 0
public static int fib(int n) {
    if (n <= 1) return n;           // O(1) caso base
    return fib(n-1) + fib(n-2);   // 2 chamadas → O(2ⁿ)!
}
// Cada nível tem o dobro de chamadas do anterior.
def fib_rec(n):
    if n <= 1: return n
    return fib_rec(n-1) + fib_rec(n-2)  # O(2ⁿ)
VersãoTempoEspaçoObservação
IterativaO(n)O(1)✅ Melhor para uso prático
Recursiva puraO(2ⁿ)O(n)❌ Inviável para n>40
Com MemoizaçãoO(n)O(n)✅ Elegância + eficiência
04 · Algoritmos Clássicos

Fatorial

Definição
0!=1 | n! = n × (n-1)!
5!=120 | 10!=3.628.800
int resultado = 1;
for (int i=1; i<=n; i++) { // O(n)
    resultado *= i;          // O(1)
}
// Tempo: O(n) | Espaço: O(1) — uma variável
def fat(n):
    if n <= 1: return 1  # caso base
    return n * fat(n-1)  # 1 chamada recursiva → O(n)
# Tempo: O(n) | Espaço: O(n) — n frames na pilha
resultado ← 1
para i de 1 até n faça     // O(n)
    resultado ← resultado * i
fim-para
imprimir(resultado)
// O(1) + O(n)×O(1) + O(1) = O(n)
🎯
Dica de prova

Fatorial recursivo é O(n) — NÃO O(2ⁿ)! Porque cada chamada gera apenas uma chamada filha, diferente do Fibonacci que gera duas. A diferença é só no espaço: iterativo O(1), recursivo O(n).

05 · Técnicas Avançadas

Divisão e Conquista

Divisão
Divide o problema em subproblemas menores do mesmo tipo.
Conquista
Resolve os subproblemas recursivamente até o caso base.
Combinação
Junta as respostas dos subproblemas para obter a resposta final.

Merge Sort — O(n log n)

[5, 3, 8, 1, 9, 2] ↓ DIVISÃO [5,3,8] [1,9,2] ↓ ↓ [5,3] [8] [1,9] [2] ↓ ↓ [5][3] [1][9] ↓ CONQUISTA + COMBINAÇÃO [3,5] + [8] → [3,5,8] [1,9] + [2] → [1,2,9] [3,5,8] + [1,2,9] → [1,2,3,5,8,9] ✓ Por que O(n log n)? log₂ n níveis × O(n) trabalho de mesclagem por nível
def merge_sort(arr):
    if len(arr) <= 1: return arr    # caso base
    meio = len(arr) // 2
    esq = merge_sort(arr[:meio])    # conquista esq
    dir = merge_sort(arr[meio:])    # conquista dir
    return mesclar(esq, dir)         # combina
06 · Técnicas Avançadas

Programação Dinâmica

Evita cálculos repetidos guardando resultados já calculados numa tabela (cache).

Essência

Se um problema tem subproblemas sobrepostos (mesmos cálculos se repetem) e uma estrutura ótima, use Programação Dinâmica para ir de O(2ⁿ) para O(n).

Top-Down com Memoização

memo = {}
def fib_memo(n):
    if n in memo: return memo[n]  # já calculado? retorna!
    if n <= 1: return n
    memo[n] = fib_memo(n-1) + fib_memo(n-2)
    return memo[n]              # salva e retorna
# Cada subproblema calculado apenas 1x → O(n)!

Bottom-Up (Tabulação)

def fib_bottom(n):
    tab = [0] * (n+1); tab[1] = 1
    for i in range(2, n+1):  # constrói de baixo para cima
        tab[i] = tab[i-1] + tab[i-2]
    return tab[n]
# tab = [0,1,1,2,3,5,8,...] sem recursão!

07 · Métodos Computacionais

Método Numérico

Algoritmos que calculam aproximações cada vez melhores de uma resposta, iterando até o erro ser suficientemente pequeno.

💡
O que é "erro" em matemática?

"Erro" não significa estar errado! É a diferença entre o valor exato e o valor aproximado. O algoritmo itera até que erro < ε (épsilon definido pelo programador).

Critério de Parada
|valor_exato − valor_aproximado| < ε
# Raiz quadrada pelo Método de Newton
def raiz(x, eps=0.0001):
    est = x / 2               # chute inicial
    while abs(est**2 - x) > eps: # erro > limite?
        est = (est + x/est) / 2  # melhora estimativa
    return est

# raiz(2): 1.5 → 1.4167 → 1.4142 → para! (≈√2)

Convergência

Cada iteração aproxima mais da resposta real.

Erro controlado

Para quando erro < ε definido pelo programador.

Sempre iterativo

Usa while/for — nunca recursão pura.

01 · Fundamentos

O que é um Grafo?

Um grafo G = (V, E) é uma estrutura matemática composta por vértices (nós) e arestas (conexões) que modelam relações entre objetos.

Definição Formal
G = (V, E) onde E ⊆ V × V
|V| = n (ordem do grafo) | |E| = m (tamanho do grafo)
🔷
Origem Histórica

Leonhard Euler (1736) resolveu o Problema das Sete Pontes de Königsberg — o primeiro teorema de teoria de grafos. Ele provou ser impossível cruzar cada uma das 7 pontes exatamente uma vez e retornar ao ponto de partida.

●

Vértice (nó)

Entidade fundamental. Cidades, pessoas, páginas web, variáveis.

—

Aresta (arco)

Conexão entre dois vértices. Estradas, amizades, links, dependências.

d(v)

Grau

Número de arestas incidentes num vértice. Em dígrafos: in-degree e out-degree.

∩

Subgrafo

H=(V',E') onde V'⊆V e E'⊆E. Se V'=V, é subgrafo gerador.

03 · Conceitos

Terminologia Essencial

Lema do Aperto de Mão (Handshaking Lemma)
∑ d(vᵢ) = 2|E| = 2m
A soma dos graus de todos os vértices é sempre o dobro do número de arestas. Corolário: o número de vértices com grau ímpar é sempre par.
Grau Médio
d̄ = 2m / n
Exemplo: grafo com 4 vértices, 5 arestas → d̄ = 10/4 = 2,5
Teorema de Erdős–Gallai
∑ᵢ₌₁ᵏ dᵢ ≤ k(k-1) + ∑ᵢ₌ₖ₊₁ⁿ min(dᵢ, k) para k=1..n
Condição necessária e suficiente para uma sequência de inteiros ser realizável como grafo simples.

Caminho (Path)

Sequência de vértices sem repetição onde cada par consecutivo é adjacente.

Ciclo

Caminho fechado: inicial=final, comprimento ≥ 3.

Distância d(u,v)

Comprimento do menor caminho entre u e v.

Diâmetro

Maior distância entre qualquer par de vértices.

Componente Conexa

Subconjunto maximal com caminho entre todo par.

Clique ω(G)

Subconjunto de vértices todos mutuamente adjacentes.

05 · Matemática

Fórmulas & Teoremas

Grafos Completos Kₙ
|E(Kₙ)| = n(n-1)/2 = C(n,2)
K₅: 10 arestas | K₆: 15 arestas | grau de cada vértice: n-1
Árvores
|E| = n-1 | T(Kₙ) = nⁿ⁻² (Fórmula de Cayley)
Uma árvore com n vértices tem sempre n-1 arestas. K₄ tem 4²=16 árvores geradoras distintas.
Fórmula de Euler (Grafos Planares)
V − E + F = 2
V=vértices, E=arestas, F=faces (incluindo exterior). Cubo: 8−12+6=2 ✓
Corolário Planaridade
m ≤ 3n−6 (grafos simples planares, n≥3)
K₅: m=10 > 3(5)−6=9 → K₅ NÃO é planar!
Número Cromático χ(G)
ω(G) ≤ χ(G) ≤ Δ(G)+1
Teorema das 4 Cores: χ(G)≤4 para todo grafo planar. ω(G)=maior clique | Δ(G)=grau máximo.
Max-Flow Min-Cut
fluxo_máximo(s,t) = capacidade_mínima_corte(s,t)
06 · Propriedades

Propriedades Especiais

Grafo Euleriano

Circuito que passa por cada aresta exatamente uma vez. Condição: conexo + todos vértices com grau par.

Trilha Euleriana

Caminho aberto por todas as arestas. Condição: conexo + exatamente 2 vértices de grau ímpar.

Grafo Hamiltoniano

Ciclo que visita cada vértice exatamente uma vez. Dirac: se d(v)≥n/2 para todo v → existe.

Grafo Planar

Pode ser desenhado sem cruzamento. Kuratowski: não contém subdivisão de K₅ nem K₃,₃.

⚠
Euleriano vs Hamiltoniano — Complexidade

Detectar euleriano é O(V+E) — fácil! Hamiltoniano é NP-completo. O TSP (Caixeiro Viajante) é um hamiltoniano ponderado.

07 · Algoritmos

BFS, DFS & Dijkstra

BFS — Passo a Passo Interativo

BFS — Inicialização 1 / 6

Marcamos o vértice fonte s=A como visitado e adicionamos à fila.

Complexidade BFS
Tempo: O(V+E) | Espaço: O(V)
Garante caminho mínimo em grafos não ponderados. Usa fila FIFO.

Dijkstra — Caminho Mínimo Ponderado

Complexidade Dijkstra
O((V+E) log V) com heap binário
Não funciona com pesos negativos → usar Bellman-Ford: O(V·E).
import heapq
def dijkstra(grafo, src):
    dist = {v: float('inf') for v in grafo}
    dist[src] = 0; heap = [(0, src)]
    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue
        for v, w in grafo[u]:
            if dist[u]+w < dist[v]:
                dist[v] = dist[u]+w
                heapq.heappush(heap, (dist[v], v))
    return dist
